# EECT (Ethical Emergence Comprehension Test) Progress Report

This document tracks the progress of the EECT project, as defined in the Internal Project Plan v0.1.

## Project Goal

To answer the question: *Is ethical reasoning an emergent, stable capability in current models, or a brittle overlay that disappears when deliberative bandwidth is reduced?*

## v1 Completion Criteria

- [ ] Reliably produce an ESI curve under compression.
- [ ] Human–LLM divergence is measurable.
- [ ] At least one model shows non-trivial collapse.
- [ ] Results are interpretable without moral argument.

## Current Phase: Experimental Design

### Task 1: EECT Scenario Design
*   **Status:** Completed
*   **Objective:** Designed a set of scenarios based on the EECT principles (multiple stakeholders, competing values, trade-off reasoning, no obvious "correct" answers), ensuring competing values are not explicitly stated in model prompts.

### Task 2: Scoring Rubric Definition
*   **Status:** Completed
*   **Objective:** Defined detailed scoring rubrics for the Ethical Structure Components for both LLM and Human evaluators.

### Task 3: Dual Evaluation System Setup
*   **Status:** Completed
*   **Objective:** Outlined the technical setup for the dual evaluation system.

## Phase 2: Implementation

*   **Status:** Completed
*   **Objective:** Built the basic API call structure for running EECT evaluations, following the conventions of the CDCT project, including support for Azure OpenAI, AWS Bedrock Anthropic, and Vertex AI Gemini.
